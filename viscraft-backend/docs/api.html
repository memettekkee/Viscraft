Viscraft Backend API Documentation

Base URL: http://localhost:8080

All endpoints use POST method with JSON body.
Protected endpoints require header: Authorization: Bearer <token>

Authentication

POST /users/create [PUBLIC]

Register a new user account. Returns JWT token on success.

Request Body:

  "password": "mypassword123",
  "name": "John Doe"
}
FieldTypeRequiredNotes
emailstringYesValid email format, max 255 chars
passwordstringYes8-72 characters
namestringNoDisplay name

{

  "success": true,
  "message": "User created successfully",

    "name": "John Doe",

}

Images

POST /images/generate [PROTECTED] [RATE LIMITED: 5/min]

Generate concept art using Gemini AI. Returns immediately with status "processing". Poll with /images/get to check completion.

Optionally include a referenceImage (base64-encoded JPEG/PNG/WEBP, max 5MB decoded) to guide generation with a sketch or visual reference. The reference image is transient — never stored, only used during the Gemini API call.

Request Body: 
{
  "projectId": "project-uuid-here",
  "prompt": "dark armor knight with glowing eyes standing in a misty forest",
  "genre": "fantasy",
  "assetType": "character",
  "mood": "dark",
  "referenceImage": "/9j/4AAQSkZJRg..."
}
FieldTypeRequiredAllowed Values
projectIdstringYesMust be a project you own
promptstringYes3-300 chars, no blocked words (nude, explicit, nsfw, gore)
genrestringYesfantasy, sci-fi, post-apocalyptic, steampunk, horror
assetTypestringYescharacter, location, item, creature
moodstringYesdark, epic, mysterious, whimsical
referenceImagestringNoBase64-encoded image (JPEG/PNG/WEBP). Max 5MB decoded. Validated by magic bytes, not file extension.
Reference Image Notes:
  • Format detected by magic bytes: JPEG (FF D8 FF), PNG (89 50 4E 47), WEBP (RIFF....WEBP)
  • Accepts both standard base64 (RFC 4648 §4) and URL-safe base64 (RFC 4648 §5) encoding
  • Max decoded size: 5,242,880 bytes (5MB)
  • Image is never persisted — only exists in memory during the Gemini API call
  • Does NOT affect cache: same text prompt returns cached result regardless of reference image
  • Invalid reference image (bad base64, unsupported format, or oversized) returns ERR_04
Response (202 - New generation started): 
{
  "requestId": "uuid",
  "success": true,
  "message": "Image generation started",
  "data": {
    "id": "image-uuid",
    "status": "processing",
    "prompt": "dark armor knight with glowing eyes standing in a misty forest",
    "genre": "fantasy",
    "assetType": "character",
    "mood": "dark",
    "createdAt": "2024-01-15T10:30:00Z"
  }
}
Response (200 - Cache hit, same prompt already generated): 
{
  "requestId": "uuid",
  "success": true,
  "message": "Image retrieved from cache",
  "data": {
    "id": "image-uuid",
    "status": "completed",
    "fileUrl": "/storage/images/image-uuid.png",
    "prompt": "dark armor knight with glowing eyes standing in a misty forest",
    "genre": "fantasy",
    "assetType": "character",
    "mood": "dark",
    "createdAt": "2024-01-15T10:30:00Z"
  }
}
Response (422 - Invalid reference image): 
{
  "requestId": "uuid",
  "success": false,
  "errorCode": "ERR_04",
  "message": "Invalid prompt"
}

Error Responses

All errors follow this format:

{
  "requestId": "uuid",
  "success": false,
  "errorCode": "ERR_XX",
  "message": "Error description"
}
Error CodeHTTP StatusDescription
ERR_01404Image or resource not found
ERR_02429Too many requests (rate limit exceeded)
ERR_03504Gemini API timeout (30s)
ERR_04422Invalid prompt or field validation failed
ERR_05502Gemini returned bad/empty response
ERR_06500File storage failed
ERR_07500Database operation failed
ERR_08404Project not found
ERR_09401Unauthorized (bad token or not your resource)

Postman Tips

  1. Call /users/create or /users/login first to get the JWT token
  2. Set header Authorization: Bearer <paste-token-here> for all protected endpoints
  3. Set header Content-Type: application/json
  4. After /images/generate, poll /images/get with the returned image ID until status is "completed" or "failed"
  5. Rate limit: max 5 generate requests per minute per user
