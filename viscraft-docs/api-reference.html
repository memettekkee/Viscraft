Ad Shots

Base path: /scenes

Async Generation Flow
POST /scenes/generate returns 202 immediately with status: "processing". The frontend polls /scenes/get every 3 seconds until status becomes "completed" or "failed". Generation typically takes 10–60 seconds.

POST /scenes/generate  PROTECTED  RATE LIMITED: 5/min

Trigger AI generation of a product ad shot. Returns 202 immediately; actual generation happens asynchronously in a background goroutine.

Request Body

FieldTypeRequiredNotes
projectIdstringYesCampaign UUID (must be owned by authenticated user)
promptstringYesRaw user description, 3–300 chars. Stored as user_prompt.
generatedPromptstringNoAssembled prompt from buildPrompt(). If provided, this is what gets sent to the AI API. Stored as generated_prompt.
referenceSceneIdstringNoUUID of an existing ad shot to use as visual reference (image-editing mode).
uploadedReferenceImagestringNoBase64-encoded JPEG/PNG/WebP, max 5MB decoded. Takes priority over referenceSceneId.

Prompt Logic

If generatedPrompt is provided, it is used as the actual AI prompt. If omitted, prompt is used directly. Both are persisted — user_prompt is the raw input the user typed, and generated_prompt is the structured prompt built by buildPrompt() on the frontend.

Response 202 (Generation Started)

{
  "requestId": "uuid",
  "success": true,
  "message": "Generation started",
  "data": {
    "id": "scene-uuid",
    "projectId": "project-uuid",
    "orderIndex": 3,
    "prompt": "A bottle of cold brew coffee",
    "status": "processing",
    "createdAt": "2025-06-15T10:30:00Z"
  }
}

Error Codes

CodeHTTPWhen
ERR_02429Rate limit exceeded (5 per minute per user)
ERR_04422Invalid prompt (too short, too long, blocked words)
ERR_08404Campaign not found or not owned by user
ERR_11422Missing required fields
ERR_13422Content policy violation (prompt rejected by AI safety filter)
Async failure codes: ERR_03 (timeout), ERR_05 (bad/empty response), ERR_13 (content policy) are set on the scene record asynchronously after the HTTP 202 response. Poll /scenes/get to detect these.

Async Generation Flow Detail

How async generation works end-to-end:
  1. Client calls POST /scenes/generate — server validates input, inserts a DB record with status="processing", and returns HTTP 202 with the new scene ID.
  2. A Go goroutine starts in the background and calls the Pollinations API (text-to-image GET or image-editing POST depending on whether a reference image was provided).
  3. Frontend receives the 202 response and begins polling POST /scenes/get every 3 seconds using SWR's refreshInterval.
  4. On AI success: goroutine saves the image to ./storage/images/{sceneId}.png and updates the DB record to status="completed" with file_url.
  5. On AI failure: goroutine sets status="failed" with the appropriate error code (ERR_03, ERR_05, or ERR_13).
  6. Next polling cycle detects status ≠ "processing" — SWR stops polling and the UI renders the final state (image or error card).
Quick Reference: Getting Started
1. Call /users/login (or /users/create) to get a JWT token.
2. Set header Authorization: Bearer <token> on all subsequent requests.
3. Call /projects/create to create a campaign.
4. Call /prompt-options to fetch available styling options.
5. Call /scenes/generate with a projectId and prompt.
6. Poll /scenes/get with the returned scene ID every 3s until status is "completed" or "failed".
